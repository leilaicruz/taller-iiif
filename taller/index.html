Taller IIIF Interactivo

Explora las posibilidades del estándar IIIF (International Image Interoperability Framework) usando Cantaloupe como servidor de imágenes y OpenSeadragon como visor.

¿Qué es IIIF? Concepto

IIIF es un conjunto de estándares abiertos para servir, compartir y anotar imágenes de alta resolución a través de la web.

🖼️ Image API

Permite solicitar partes específicas de una imagen con parámetros en la URL: región, tamaño, rotación, calidad y formato.

📖 Presentation API

Define cómo organizar imágenes en secuencias (manifiestos), como las páginas de un libro o las piezas de una colección.

🔍 Content Search API

Permite buscar texto dentro de los recursos IIIF, como texto OCR en páginas de documentos digitalizados.

🏗️ Nuestra infraestructura

Cantaloupe 5.0 — Servidor IIIF de imágenes escrito en Java. Soporta IIIF Image API 2.1 y 3.0, múltiples formatos, cache, y procesamiento dinámico.

{scheme}://{host}/iiif/3/{imagen}/{region}/{size}/{rotation}/{quality}.{format}

OpenSeadragon — Visor JavaScript de alta resolución. Permite zoom profundo, navegación fluida, y consume directamente endpoints IIIF.

OpenSeadragon({
  tileSources: "/iiif/3/1.jpg/info.json"
});

Anatomía de una URL IIIF Image API 3.0

Cada parte de la URL controla un aspecto de la imagen que se devuelve. Pasa el cursor sobre cada parte.

{server}/iiif/3 / 1.jpg / pct:25,25,50,50 / 800, / 0 / default .jpg
Base del servidor Identificador Región Tamaño Rotación Calidad Formato

Valores de Región

full            → Imagen completa
square          → Recorte cuadrado centrado
x,y,w,h         → Píxeles (ej: 100,100,500,300)
pct:x,y,w,h     → Porcentaje (ej: pct:25,25,50,50)

Valores de Tamaño

max             → Tamaño máximo disponible
w,              → Ancho fijo, alto proporcional
,h              → Alto fijo, ancho proporcional
w,h             → Tamaño exacto (puede distorsionar)
!w,h            → Encajar en caja (mantiene proporción)
pct:n           → Porcentaje del original

Transforma la imagen a diferentes calidades (color, grises, bitonal) y formatos (JPG, PNG, WebP).

info.json Metadatos

Cada imagen IIIF expone un endpoint info.json con sus metadatos: dimensiones, formatos soportados, niveles de tile, etc. Es lo que usa OpenSeadragon para saber cómo pedir los tiles.

¿Qué contiene info.json?

{
  "@context": "http://iiif.io/api/image/3/context.json",
  "id": "{server}/iiif/3/1.jpg",
  "type": "ImageService3",
  "protocol": "http://iiif.io/api/image",
  "width": 4000,      ← Ancho en píxeles
  "height": 3000,     ← Alto en píxeles
  ...
}

Los campos clave son:

  • width / height — Dimensiones reales
  • tiles — Configuración de tiling para zoom
  • sizes — Tamaños prefabricados disponibles
  • profile — Capacidades del servidor
  • extraFormats — Formatos adicionales soportados
  • extraQualities — Calidades disponibles

Zoom progresivo Deep Zoom

IIIF permite hacer zoom progresivo pidiendo regiones cada vez más pequeñas a resolución completa. Así funciona el "deep zoom" que permite explorar imágenes gigantes.

¿Cómo funciona el tiling?

Cuando una imagen es muy grande, el servidor la divide en tiles (baldosas) de ~256×256 o 512×512 píxeles. El visor (OpenSeadragon) solo pide los tiles que necesita para el nivel de zoom y la zona visible, haciendo que imágenes de cientos de megapíxeles sean navegables en el navegador.

Nivel 0:  1 tile (thumbnail)
Nivel 1:  4 tiles (2×2)
Nivel 2:  16 tiles (4×4)
Nivel 3:  64 tiles (8×8)
...
Nivel N:  resolución completa, solo carga los tiles visibles

Presentation API IIIF Presentation 3.0

La Image API sirve imágenes individuales. La Presentation API organiza esas imágenes en estructuras con significado: libros, manuscritos, exposiciones, colecciones.

🧩 Conceptos clave

Manifest — Describe un objeto digital completo (un libro, una obra). Contiene metadatos y una secuencia de Canvas.

Canvas — Una superficie abstracta (como una página) donde se "pintan" contenidos mediante anotaciones.

Annotation — Asocia contenido (imagen, texto, audio) a un Canvas o zona de un Canvas.

Collection — Agrupa múltiples Manifests jerárquicamente.

Range — Define una estructura lógica (capítulos, secciones) dentro de un Manifest.

📐 Anatomía de un Manifest

{
  "@context": "http://iiif.io/api/presentation/3/context.json",
  "id": "https://ejemplo.com/manifest.json",
  "type": "Manifest",
  "label": { "es": ["Mi libro"] },
  "metadata": [...],        ← Metadatos descriptivos
  "behavior": ["paged"],    ← Comportamiento del visor
  "items": [                ← Array de Canvas
    {
      "type": "Canvas",
      "label": { "es": ["Página 1"] },
      "width": 1000, "height": 1500,
      "items": [{            ← AnnotationPage
        "items": [{          ← Annotation (painting)
          "body": {          ← La imagen IIIF
            "service": [{ "type": "ImageService3" }]
          }
        }]
      }]
    }
  ]
}

Visor Mirador Presentation API Viewer

Mirador es el visor IIIF más completo. Lee manifiestos y permite navegar páginas, ver metadatos, comparar imágenes lado a lado, y gestionar anotaciones.

Funciones de Mirador

  • Navegación de páginas — Thumbnails, índice, avance/retroceso
  • Multi-ventana — Abre varios manifiestos a la vez
  • Metadatos — Panel con información descriptiva del manifest
  • Anotaciones — Visualiza y crea anotaciones sobre las imágenes
  • Zoom profundo — Integración con IIIF Image API para deep zoom
  • Comparación — Compara páginas o manifiestos lado a lado

Anotaciones IIIF Web Annotation

Las anotaciones IIIF siguen el estándar W3C Web Annotation y permiten asociar contenido (texto, tags, enlaces) a zonas específicas de un canvas.

Tipos de motivación

painting — Contenido principal del canvas (la imagen)

commenting — Comentarios sobre una zona

tagging — Etiquetas o categorías

describing — Descripción detallada

linking — Enlaces a recursos relacionados

supplementing — Contenido complementario (OCR, transcripción)

Target con fragmento

// Zona rectangular: x, y, ancho, alto
"target": "...canvas/1#xywh=100,200,500,300"

// Toda la superficie del canvas
"target": "...canvas/1"

// Porcentaje del canvas
"target": "...canvas/1#xywh=pct:10,10,80,80"

El fragmento #xywh= usa el estándar W3C Media Fragments para señalar regiones rectangulares.

Ejemplo de anotación completa

{
  "id": "https://ejemplo.com/anno/1",
  "type": "Annotation",
  "motivation": "commenting",
  "body": {
    "type": "TextualBody",
    "value": "Este detalle muestra la técnica del claroscuro.",
    "format": "text/plain",
    "language": "es"
  },
  "target": "https://ejemplo.com/canvas/1#xywh=250,100,400,300"
  ↑ Señala una zona rectangular de 400×300 px
    empezando en (250, 100) del canvas
}

🔎 IIIF Content Search API Búsqueda dentro de documentos

La Content Search API permite buscar texto dentro de documentos IIIF que tienen OCR (reconocimiento óptico de caracteres). Los resultados devuelven las coordenadas exactas donde aparece cada término en cada página.

📖 ¿Cómo funciona?

📄

1. Manifiesto con OCR

El manifiesto IIIF declara un service de tipo SearchService1 que apunta al endpoint de búsqueda.

🔍

2. Consulta HTTP

Se envía una petición GET con ?q=término al endpoint. El servidor busca en todo el texto OCR del documento.

📍

3. Coordenadas xywh

Cada resultado incluye el canvas, la posición #xywh=x,y,w,h y el texto encontrado con contexto.

🏗️ Estructura del servicio en el manifiesto

Así se declara un servicio de búsqueda dentro de un manifiesto IIIF (Presentation API 2.1):

{ "@context": "http://iiif.io/api/search/0/context.json", "@id": "https://example.org/search/v0/{manifest-id}", "profile": "http://iiif.io/api/search/0/search", "label": "Search within this manifest", "service": { "@id": "https://example.org/search/autocomplete/v0/{manifest-id}", "profile": "http://iiif.io/api/search/0/autocomplete", "label": "Autocomplete" } }

📐 Estructura de la respuesta

  • @type: sc:AnnotationList
  • within.total: número total de resultados
  • resources[]: array de anotaciones
  • resources[].resource.chars: texto encontrado
  • resources[].on: canvas + #xywh=x,y,w,h
  • hits[]: agrupación por contexto
  • hits[].match: texto exacto del match
  • hits[].before / .after: texto de contexto

💡 Casos de uso

  • 📚 Búsqueda full-text en libros digitalizados
  • 📰 Búsqueda en periódicos históricos

  • 🔬 Buscar términos científicos en publicaciones
  • 🏛️ Buscar referencias en archivos históricos
  • ✏️ Resaltar resultados sobre la imagen original
  • 🔗 Deep-linking a posiciones exactas en un canvas
